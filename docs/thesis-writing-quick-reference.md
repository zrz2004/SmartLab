## 🎯 AIGC 率控制的 5 分钟速查

| 写作场景 | ❌ 错误做法 | ✅ 正确做法 |
|---------|----------|----------|
| 描述技术架构 | "BLoC是一种状态管理模式..." | "项目在 `features/dashboard/presentation/bloc` 中实现了 DashboardBloc，包含..."  |
| 解释设计决策 | "我们选择MQTT是因为它性能好" | "相比REST轮询（延迟>1分钟），MQTT 具有<1秒的实时性，同时支持离线缓存..." |
| 数据对比 | "系统具有良好性能" | "平均报警延迟 1.2 秒（目标<2秒，达成度100%）" |
| 列举功能 | "支持多个功能" | 用表格列出具体的4层角色×11项权限矩阵 |
| 总结特点 | "架构易维护" | "新增设备类型仅需添加5个文件（见4章模块设计）" |

## 📚 各章节的核心内容与原创配比

```
第1章（绪论）
├─ 1.1 研究背景：60% 项目场景 + 40% 文献
├─ 1.2 研究意义：70% 项目数据 + 30% 文献
├─ 1.3 相关工作：40% 文献 + 60% 项目差异分析
└─ 1.4 内容结构：100% 原创（论文提纲）

第2章（相关工作与技术基础）
├─ 2.1-2.3 技术理论：50% 文献 + 50% 项目案例
└─ 2.4 权限管理：30% 文献 + 70% 项目实现

第3章（需求与架构）★ 原创性最强
├─ 3.1 业务需求：80% 项目调研 + 20% 文献标准

└─ 3.3 技术选型：85% 项目实现 + 15% 文献对比

第4章（核心实现）★ 代码密集章节
├─ 每个模块：95% 项目代码展示 + 5% 文献理论

├─ 测试结果：100% 项目测试数据

第6章（总结与展望）
```

## 🔑 每一段的必要元素检查

写完每个段落，逐条检查（打勾则进行下一段）：

### 对于"概念讲解"段落
- [ ] 引用了项目代码或文件名吗？（如 `smart_lab/lib/features/...`）
- [ ] 给出了具体的设计例子或参数吗？（数字、阈值、配置）
- [ ] 解释了**为什么**这样设计而不是其他方案？
- [ ] 避免了"BLoC 是一种..."这样的通用句式？

### 对于"对比分析"段落
- [ ] 列举了至少 3 个可选方案吗？
- [ ] 用表格或项目符号对比了各方案的优劣？
- [ ] 明确说明了项目场景的约束条件（多数据源、权限复杂等）？
- [ ] 给出了具体的选择理由（而非"性能好"这样的模糊说法）？

### 对于"实现细节"段落
- [ ] 贴了项目代码片段（3-15 行）？
- [ ] 标注了文件路径和行号范围？
- [ ] 用自己的语言解释了代码的目的（而非逐行翻译）？
- [ ] 说明了这段代码解决的是什么问题？

### 对于"数据展示"段落
- [ ] 用了表格而非长句子列举？
- [ ] 数字都能在项目文档或代码中找到出处？
- [ ] 表格有题注和单位说明？
- [ ] 表格后有文字说明其含义和启示？

## 💾 写作工作流（日程）

### 每天的写作流程

**上午（2-3小时）**：
1. 打开**详细大纲**（thesis-detailed-outline.md）
2. 选择一个 2.1 节或 3.2 节这样的"小节"
3. 按照大纲的"核心论述内容"快速铺开 800-1200 字
4. *不* 追求完美，只追求把信息铺出来

**下午（1-2小时）**：
1. 打开**写作规范指南**（thesis-writing-standards-and-aigc-control.md）
2. 逐段检查是否有"AIGC陷阱"
3. 用项目代码、表格、具体数字替换模糊表述
4. 添加必要的代码片段、图表、表格

**晚上（0.5-1小时）**：
1. 阅读刚才写的段落
2. 检查逻辑是否连贯、论据是否充分
3. 记录"还缺什么信息"的清单（用于查找文献或项目文档）

### 周的里程碑

| 周 | 目标 | 产出 |
|----|------|------|
| 第1周（4.21-4.27） | 第1-2章 + 第3章1-2节 | 3500字 |
| 第2周（4.28-5.4） | 第3章3节 + 第4章1-2节 | 4000字 |
| 第3周（5.5-5.11） | 第4章3-4节 + 第5章1-2节 | 3500字 |
| 第4周（5.12-5.18） | 第5章3节 + 第6章 | 2000字 |
| 第5周（5.19-5.25） | 修改、AIGC检测、参考文献补充 | 定稿 |

## 📖 论文中常用的"过渡句"模板

用于连接段落和章节：

**从理论到实现**：
- "基于上述设计，项目在 `smart_lab/lib/...` 中实现了..."
- "将这一理论应用到实验室监测场景，本项目采取了..."

**从现状到改进**：
- "传统方案存在的问题是...，本项目的改进是..."
- "相比现有研究，本项目在以下方面的创新包括..."

**从总体到细节**：
- "系统总体架构包括三层：... 其中，第...层的关键设计是..."

**从问题到解决**：
- "当遇到...问题时，项目采取的解决方案是..."

**从一个模块到下一个模块**：
- "完成了...的设计后，还需要考虑...模块的实现..."

## 🔗 项目关键文件速查

当你需要引用项目的具体实现时：

| 内容 | 文件位置 | 用途 |
|------|---------|------|
| 需求与权限 | `docs/USER_PERMISSION_DESIGN.md` | 用户角色、权限矩阵 |
